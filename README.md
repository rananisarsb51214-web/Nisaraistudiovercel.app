# 🦾 NisarAI-Studio-v2: Autonomous Multi-Agent & Monetization Substrate

NisarAI-Studio-v2 is an enterprise-grade, cloud-native software architecture running on **Next.js 15 (App Router)** and **Node.js 22 Runtime**. It is engineered to orchestrate self-grounding AI agents, automate high-speed market research via real-time tool calling, and execute aggressive traffic monetization models optimized for high-yield digital enterprise.

---

## 🏗️ System Architecture Matrix

The entire repository infrastructure is segmented into four interconnected architectural layers:

1. **Phase 1: Semantic SEO Substrate:** Server-Side Hard Grounding.
   Injects cryptographic JSON-LD structured data mapping into the Next.js layout engine, identifying the entire architecture as an authoritative `SoftwareApplication` to search engine crawlers.

2. **Phase 2: Distributed Cloud Core:** Node.js 22 & Firebase Admin.
   Establishes secure service account handshakes with the backend Firebase Admin SDK. Enforces cryptographic update validation loops inside Firestore security rules for campaign matching.

3. **Phase 3: Autonomous Multi-Agent Chainer:** Gemini Core & Function Calling.
   Orchestrates a parallelizable system of three specialized execution agents (Research Agent, Market Validation Agent, and Elite Copywriter Agent) backed by native browser-scraping function calls.

4. **Phase 4: Aggressive Revenue Injection Loop:** Traffic Arbitrage & Geo-Targeting.
   Injects AdSense credentials (`pub-8918609159122844`) programmatically. Evaluates real-time edge geo-location traffic to execute cloaked redirects and launch high-ticket conversion popups for overseas users.

## 📂 Core File Tree & Infrastructure Mapping

| File Path | Functional Substrate | Operational Priority |
| --- | --- | --- |
| `src/app/layout.tsx` | Server-side HTML layout and SEO schema injection | `CRITICAL_INITIALIZATION` |
| `src/lib/firebase-admin.ts` | Stateless initialization of the Firebase Admin SDK substrate | `INFRASTRUCTURE_CORE` |
| `src/lib/agents/orchestrator.ts` | Grounded Multi-Agent configuration & Gemini SDK mapping | `BRAIN_ORCHESTRATION` |
| `src/lib/agents/browserAgent.ts` | Target URL scraper, trend parser, and competitor discovery tool | `LIVE_TOOL_CALLING` |
| `src/lib/agents/workflowChainer.ts` | Sequential execution manager linking Agent 1, 2, and 3 | `PIPELINE_AUTOMATION` |
| `src/lib/monetization/injector.ts` | Automated insertion of AdSense codes and tracking scripts | `MONETIZATION_LOOP` |
| `src/lib/monetization/arbitrageMasker.ts` | Edge-level bot filtration and high-ticket traffic vectoring | `REVENUE_ARBITRAGE` |
| `src/components/monetization/ConversionModal.tsx` | Intent-delayed client pop-up optimized for overseas luxury conversion | `HIGH_TICKET_CONVERSION` |
| `src/app/ads.txt/route.ts` | Programmatic crawler verification for Google AdSense compliance | `CRAWLER_VALIDATION` |

## 🚦 Verification & Deployment Vectors

### 1. Environment Substrates (.env.production)
Ensure the following cryptographic strings are securely bound to your Vercel or Google Cloud Run runtime node:
- `GEMINI_API_KEY`: Elite generative model authentication token.
- `FIREBASE_PRIVATE_KEY` / `FIREBASE_CLIENT_EMAIL`: Enterprise admin service credentials.

### 2. Runtime Limits (vercel.json)
Serverless edge functions mapped under `src/app/api/agent/**/*.ts` are pre-allocated a `maxDuration: 60` execution envelope to prevent timeouts during long-form Multi-Agent chaining loops and strategic audio briefing renderings.
